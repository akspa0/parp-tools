# Product Context — AlphaWDTAnalysisTool

## Why this exists
- Convert WoW Alpha WDT/ADT content into WotLK-compatible ADTs with high fidelity.
- Patch LK `MCNK.AreaId` per chunk using an explicit, auditable mapping generated by DBCTool.V2.
- Keep conversions deterministic and avoid heuristics in the patching path.

## How it should work
- Input: Alpha WDT/ADT, community listfile, optional LK listfile, and DBCTool.V2 `compare/v2/` outputs.
- Output: WotLK ADTs/WDT and per-tile verify CSVs when verbose.
- Strict CSV-only mapping:
  - Use DBCTool.V2 per-source-map numeric crosswalks `(src_mapName, src_areaNumber) -> tgt_areaID`.
  - No name-based or zone-base fallbacks; if there is no explicit CSV row, write `0`.
  - Accept any non-zero per-map row; prefer `via060` rows when both via and non‑via exist.

## User experience goals
- Simple CLI: point to a WDT and a `compare/v2/` directory and get patched ADTs.
- Strong guarantees: never invent targets; only apply what the CSVs specify.
- Easy to audit: verbose runs emit per-tile verify CSVs.
